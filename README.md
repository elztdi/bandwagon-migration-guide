## 搬瓦工迁机房是什么：先把概念说清楚

搬瓦工迁机房，对应的是 KiwiVM 控制面板里那个叫 **"Migrate to another DC"** 的功能。DC 是 Data Center（数据中心）的缩写。

简单说就是：把你当前跑在某个机房的 VPS，物理搬到另一个机房去，数据完整保留，停机时间一般只有几秒到几分钟。迁完之后你会收到一封邮件通知，里面有新的 IP 地址。

这个功能不收钱，但有两个实际消耗需要知道：一是每次迁移**会占用当月流量**（迁移的流量和 VPS 存储内容大小有关），二是**IP 地址会变**——新机房给你分配新 IP，原来的 DNS 记录、软件配置里的 IP 绑定都要相应更新。

免费不限次，但也别没事乱迁，一次迁移要吃掉一部分月流量。

## 搬瓦工迁机房操作步骤（5 步完成）

操作本身真的不难，照着做就行。

1. **登录 KiwiVM 控制面板**
   进官网后台，找到你要迁移的 VPS，点击对应的 KiwiVM Control Panel 按钮进入。每台 VPS 的 KiwiVM 入口在 Services → My Services 里。

2. **点击左侧菜单"Migrate to another DC"**
   这个选项在左边导航栏，点进去之后页面会显示你当前所在机房，以及可以选择迁往的目标机房列表。

3. **选择目标机房，点击"Confirm on next step"**
   从列表里选你想去的机房，点确认进下一步。目标机房以后台实时显示为准，不是所有机房都始终可选，有时候某个节点满了就不在列表里。

4. **点击"Start Migration"开始迁移**
   下一页会展示新机房的 IP 地址预览，确认没问题就点开始。迁移期间 VPS 继续运行，只在最后切换阶段停机几秒到几分钟。

5. **等邮件通知，更新 IP 配置**
   迁移一般 10-15 分钟完成，完成后会发邮件告知新 IP。如果你的 VPS 上有网站或服务，记得去更新 DNS 解析和配置文件里写死的 IP 地址。

整个流程不需要找客服，完全自助。用过一次就不觉得复杂了。

## 迁机房三个必须提前知道的事

**一、IP 一定会变**

这是最容易被忽略的。迁到新机房之后，IP 地址肯定是新的，原来那个 IP 不再属于你。如果你有域名绑着、反代配置写着 IP、或者服务绑定了固定 IP——迁完得全部手动更新。

顺便说一下：如果你是想通过迁机房来解决 IP 被墙的问题，这条路目前走不通了。搬瓦工的系统会在迁移前自动检测当前 IP 状态，检测到 IP 被墙的话，迁移功能直接被禁用，会报错 734152。IP 被墙唯一的解法是先付费换 IP，确认 IP 可用之后再考虑迁机房。

**二、迁移本身会消耗流量**

每次迁机房都会占用月流量，具体消耗量跟 VPS 磁盘里的存储内容大小正相关。VPS 越干净（数据少），迁移消耗的流量越少，也越快。如果你临近月底流量快用完，最好等流量重置再迁。

**三、迁完建议测速再用**

换到新机房之后，先用 ping 测一下延迟，再跑个速度，确认线路对你确实更好。不合适的话还可以迁回去，甚至再换别的——次数不限。

## 迁移失败怎么办：常见报错原因和处理方法

**报错：Migration backend is currently not available for this VPS. (734152)**

出现这个提示有两种情况：一是当前迁移的人太多、系统繁忙，等 10-15 分钟重试通常能解决；二是你的 IP 已经被墙，这种情况等多久都没用，先去检测 IP 状态，确认 IP 正常之后再试。

IP 状态检测方法：登录 KiwiVM 后台，找到 IP 相关的检测工具，或者用第三方多地 Ping 工具（如 ping.pe）确认连通性。

**报错：Region is full. Please try again in a few hours. (734104)**

目标机房满了，等几小时再试，或者换个目标机房。这种情况跟你 VPS 状态无关，纯粹是那个节点当时没位置了。

**后台找不到"Migrate to another DC"这个菜单**

说明你的套餐本身不支持迁机房，或者是单机房限定方案。回去看购买时的套餐说明，部分限量版套餐不带迁机房功能。

## FAQ：搬瓦工迁机房常见问题

**Q：迁机房会丢数据吗？**

不会。官方明确说明迁移过程中所有数据完整保留，停机时间只在最后切换阶段的几秒到几分钟。建议迁之前做个快照备份以防万一，但实际上迁移丢数据的情况很少见。

**Q：迁机房要花钱吗？次数有没有限制？**

不收钱，次数没有限制。但每次迁移会消耗月流量，消耗多少取决于 VPS 上存储了多少数据。

**Q：迁机房之后原来的 IP 还能用吗？**

不能。迁到新机房之后 IP 肯定会换，原来的 IP 不再属于你。如果有服务依赖固定 IP，迁完要记得更新所有配置。

**Q：迁移大概要多长时间？**

一般 10-15 分钟，具体时间跟 VPS 上的文件数量和大小有关。数据少的话可能更快，完成后会收到邮件通知。

**Q：KVM 套餐迁到 CN2 机房，流量真的会缩减吗？**

是的。KVM 套餐迁到 DC3（CN2 机房）之后，月流量变成原套餐的 1/3。举个例子，原来每月 1TB 流量，迁过去之后变成大约 333GB。这点很重要，选机房前要算清楚。

**Q：我想通过换机房来解决 IP 被墙，可以吗？**

不可以。搬瓦工的系统会在迁移前检测 IP 状态，IP 被墙的话迁移功能会直接禁用。唯一的办法是先付费购买新 IP，等 IP 恢复正常再操作迁机房。
